关于HTTP相关的知识真的是一听就困…

想当然的认为不就是一个协议嘛…

但是弄不懂HTTP真的很吃亏

什么是HTTP？

HTTP是一种通信协议，计算机之间要共同遵守这个规则才能够彼此之间相互通信

  • http客户端发起请求，创建端口
  • http服务器在端口监听客户端请求
  • http服务器向客户端返回状态和内容

HTTP走过的环节？

以www.imooc.com为例，在Chrome搜索

  1. 第一步，域名解析。

    • Chrome搜寻自身的DNS缓存（Chrome://net-internals/#dns）

    • 当浏览器没有找到缓存或者缓存已经失效时，搜索操作系统自身的DNS缓存

    • 若操作系统也没有找到缓存，则读取本地的HOST文件

    • 若本地的HOST文件也没有找到缓存，浏览器发起一个DNS的一个系统调用，向本地主库DNS服务器（一般由宽带运营商提供）发起一个域名解析请求

      • 宽带运营商服务器查看本身缓存

      • 运营商服务器发起一个迭代DNS解析请求，首先去找根域的DNS的IP地址，返回com域的顶级域的IP地址，com域的顶级域的服务器返回imooc.com域的IP地址，imooc.com域的DNS服务器，返回imooc.com的域名对应的IP地址

      • 运营商服务器把结果返回操作系统内核同时缓存起来

      • 操作系统内核把结果返回浏览器

      • 最终浏览器拿到了www.imooc.com对应的IP地址

  2. 浏览器获得域名对应的IP地址后，发起HTTP的“三次握手”

    • 浏览器以一个随机端口向服务器的Web程序(如80端口)发起TCP连接请求，这个连接请求通过层层的路由设备到达了服务器端以后进入到了网卡，然后进入到内核的TCP/IP协议栈，还有可能要经过防火墙的过滤，最终到了Web服务器，建立起了TCP/IP的连接
    • 客户端向服务器发起请求：”嘿老哥你能听到我说话吗？“
    • 服务器：”嘿小老弟我能听到你说话“
    • 客户端：”嘿那我开始说话啦“
    • 握手成功！

  3. TCP/IP连接建立起来后，浏览器就可以向服务器发送HTTP请求了

  4. 服务器端接收到了这个请求，根据路径参数，经过后端的一些处理之后，把处理后的一个结果参数返回给浏览器，如果是慕课网的页面就会把完整的HTML页面代码返回给浏览器

  5. 浏览器拿到了完整的HTML页面代码，在解析和渲染这个页面的时候，里面的JS、CSS、图片静态资源，同样也是一个个的HTTP请求，都需要经过以上步骤。

  6. 浏览器根据拿到的资源对页面进行渲染，最终把一个完整的页面呈现给了用户
